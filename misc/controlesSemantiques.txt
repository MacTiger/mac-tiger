Contrôles sémantiques :
	- Division par zéro (en clair).
	- Division par zéro (géré dynamiquement dans le code généré).
	- Dépassement d'Array (en clair).
	- Dépassement d'Array (géré dynamiquement dans le code généré).
	- L'index d'un subscript (Array) doit être entier à sa définition ET pour tout accès à ses valeurs.
	- Variable utilisée dans sa portée de déclaration.
	- Une négation ne porte que sur un INT, son résultat est un INT.
	- On entre le bon nombre d'arguments à une fonction
	- Les types des arguments passés en paramètre correspondent
	- Si on a défini un type de retour d'une fonction, il faut que l'on retourne un objet de même type. En tenir compte pour une éventuelle affectation.
	- Si on définit un type, ne pas lui attribuer des valeurs dont il n'a pas été fait mention lors de la déclaration. Cf tests/fail/structureFiled


Pour infixExp, pour les opérateurs :
	- "+", "-", "*", "/" : les opérandes (et le résultat) sont de type INT
	- "=","<>" : les opérandes sont de même type, le résultat est un INT
	- ">","<","<=",">=" : les opérandes sont de même type, le résultat est un INT
