# Contrôles sémantiques

Ceux qui sont cochés ont un test écrit dans `tests/fail/semantic/`. Ils n'ont pas forcément été écrit en Java.

- [x] Division par zéro (en clair)
- [x] Division par zéro (calculé)
- [x] Dépassement d'Array (en clair)
- [x] Dépassement d'Array (calculé)
- [x] L'index d'un subscript (Array) doit être entier à sa définition ET pour tout accès à ses valeurs.
- [] Variable utilisée dans sa portée de déclaration.
- [x] Le programme ne contient pas de double déclaration
- [x] Une négation ne porte que sur un INT, son résultat est un INT.
- [x] On entre le bon nombre d'arguments à une fonction
- [x] Les types des arguments passés en paramètre correspondent
- [x] Si on a défini un type de retour d'une fonction, il faut que l'on retourne un objet de même type. En tenir compte pour une éventuelle affectation.
- [] Si on définit un type contenant des champs, on ne peut pas chercher à modifier un champ n'ayant pas été déclaré. Cf tests/fail/structureFiled
- [x] "+", "-", "*", "/" : les opérandes (et le résultat) sont de type INT
- [x] "=","<>" : les opérandes sont de même type, le résultat est un INT
- [x] ">","<","<=",">=" : les opérandes sont de même type, le résultat est un INT
- [x] On ne peut pas déclarer deux types en les assignant l'un à l'autre
- [] Un dictionnaire ne peut avoir deux champs portant le même nom
- [x] On ne peut pas déclarer une variable en lui assignant une variable déclarée après
- [] Une fonction ne peut pas avoir de paramètres ayant le même nom
